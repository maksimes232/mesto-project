+Новая карточка добавляется в конец контейнера с карточками, а должна всегда добавляться в начало.

Открывая попап для создания карточки пользователь всегда должен видеть данный попап в обновленном состоянии. 
 Сейчас же если я введу данные и закрою этот попап, и потом открою его снова я увижу те данные которые вводил, 
 а должен видеть пустые поля.

+Если остается менее 3 карточек, оставшиеся карточки изменяют свой размер и становятся шире. 
 Если остается одна карточка, то она становится еще шире. 
 Это неверно. Размер карточек должен быть постоянен и не зависеть от общего количества карточек.

+Файл script.js следует поместить в папку src

+В приложении есть 3 попапа. Все попапы должны быть изначально созданы в index.html. Каждый попап это по сути div. 
 Данный div обязательно должен иметь общий для всех попапов класс - popup, дополнительно к этому 
 он должен иметь класс/id относящийся к конкретному попапу - popup_edit-profile, popup_add-card, popup_image. 
 Именно по этим уникальным селекторам мы должны выполнять поиск элементов попапа.

+Для открытия/закрытия попапов следует создать универсальные функции, которые имеют общий для всех попапов функционал
 -  openPopup(popup) и closePopup(popup). 
 В виде аргумента данные функции должны принимать элемент попапа. 
 Если какому-то попапу не хватает функционала универсальной функции, тогда следует создать функцию 
 обертку вокруг универсальной, которая будет расширять её функционал.

По названию переменной должно быть понятно, что конкретно в ней храниться.
 Например, popup - некорректное название, корректное название popupEditProfile и тд.
 Название переменной, которая хранит DOM элемент должно начинаться с имени существительного и маленькой буквы.
 Название переменной не должно содержать имен числительных. Название переменной должно соответствовать
 правилу camelCase.
 Следует просмотреть все названия переменных и внести необходимые изменения.

По названию функции должно быть понятно, что конкретно она делает.
 Название функции должно начинаться с глагола (в отличии от названия обычной переменной)  и маленькой буквы.
 Например, formSubmit - некорректное название, корректное название submitEditProfileForm, render -
 некорректное название, корректное название renderInitialCards и тд.
 Название функции не должно содержать имен числительных. Следует просмотреть все названия функци и
 внести необходимые изменения.

Функционал создания карточки следует вынести в функцию createCard ,
 она должна просто возвращать карточку с установленными на её элементы слушателями для установки лайка,
 удаления карточки и открытия попала с изображением Данная функция не должна добавлять карточку в DOM.
 Это нужно для того, чтобы избежать дублирования кода при создании карточки через попап и при отрисовке
 изначальных карточек.
